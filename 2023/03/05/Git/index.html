Git原理

init是对本地仓库的操作

add是对缓存区的操作 （添加）

commit是对本地仓库的操作 （添加）

clone是对远程仓库的操作 （克隆）

graph LR;
A((工作区))==add==>B((暂存区/缓存区))
B==commit==>C((版本库或本地仓库))
C==push==>D((版本库或本地仓库))
C==checkout==>A
D==fetch/clone==>C
D==pull==>A

全局设置

#配置远程仓库的用户
git config --global user.name "myname"
git config --global user.email "myEmail"

#查询远程仓库的用户
git config --global user.name
git config --global user.email

#删除远程仓库的用户
git config  --global --unset user.name "myname"
git config --global --unset user.email "myEmail"

#增加远程仓库的用户
git config  --global --add user.name "myname"
git config  --global --add user.email "myEmail"

创建git仓库

仓库不存在

mkdir yourDir   							#创建yourDir文件夹
cd yourDir									#转到yourDir文件夹
git init 									#初始化yourDir文件夹为git文件夹
touch README.md								#产生一个空的README.md文件
git add README.md							#添加README.md文件
git commit -m "first commit"				#添加注释
git remote add origin https://gitee.com/emailName/yourDir.git		#连接远程仓库
git push -u origin "master"					#上传到master分支

仓库若存在

cd existing_git_repo						
git remote add origin https://gitee.com/emailName/yourDir.git		#连接远程仓库
git push -u origin "master"					#上传到master分支

克隆仓库

git clone 远程仓库Url     #把远程仓库的内容克隆到本地

提交与修改

git add   #添加文件到暂存区
git status #查看仓库当前的状态，显示有变更的文件。
git diff  #比较文件的不同，即暂存区和工作区的差异。
git commit #提交暂存区到本地仓库。
git reset #回退版本。
git rm #将文件从暂存区和工作区中删除。强制前面加-f
git mv #移动或重命名工作区文件。

日志

git log #查看历史提交记录
git log  --oneline  #查看简洁版本的历史记录
git log --reverse #逆向查看
git log --author #只想查找指定用户的提交日志


git blame <file>  #以列表形式查看指定文件的历史修改记录

远程操作

git remote   	#远程仓库操作
git fetch		#从远程获取代码库
git pull		#下载远程代码并合并
git push		#上传远程代码并合并

分支管理

git branch (branchname)  #创建分支
git branch      #查看分支

git checkout (branchname) #切换分支
git checkout -b (branchname) #创建分支并切换到该分支
git checkout -d (branchname) #删除该分支
git merge 					#合并分支

文件

git blame <file>  #查看文件的修改记录
